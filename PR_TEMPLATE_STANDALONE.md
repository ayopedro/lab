# Pull Request Template (Standalone)

Use this generic template in any repository (copy/paste). It is decoupled from project-specific scripts.

## 1. Summary
Brief overview of the change and its purpose.

## 2. Related Issues / Tickets

- Relates to #

## 4. Description of Changes
List main changes:
- Added ...
- Updated ...
- Removed ...

## 5. Implementation Notes / Decisions
Explain key decisions, trade-offs, or patterns chosen.

## 6. Testing Performed
| Test | Result |
|------|--------|
| Unit tests | PASS / FAIL |
| Integration tests | PASS / FAIL |
| Manual verification | description |
| Security checks | e.g., dependency scan |

## 7. Screenshots / Logs (Optional)
Include before/after screenshots or terminal logs if UI/CLI changes.

## 8. Breaking Changes
Does this introduce breaking changes? If yes, describe migration steps.

## 9. Rollback Plan
If this needs to be reverted, how?
1. Revert commit SHA
2. Restore previous config
3. Notify affected users

## 10. Security / Privacy Considerations
Any secrets, auth flows, or data handling changes? Confirm no sensitive info committed.

## 11. Performance Considerations
Any benchmarks or expected impact on resource usage.

## 12. Documentation Updates
- [ ] README

- [ ] API docs / comments

## 13. Checklist
- [ ] Follows project coding standards
- [ ] Adds/updates tests where relevant
- [ ] Commits follow Conventional Commits
- [ ] No secret or sensitive data introduced
- [ ] Linting passes locally
- [ ] CI green

## 14. Open Questions
List unresolved questions for reviewers.

## 15. Additional Notes
